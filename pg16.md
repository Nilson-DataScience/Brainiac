# Declarando Variáveis

**`let`** — Permite declarar variáveis com escopo de bloco, reduzindo o risco de erros por uso indevido em diferentes partes do código.

**`const`** — É similar, mas cria variável imutável, ideal para valores que não devem ser alterados após a atribuição inicial.

> 💡 **Complemento — e o `var`?** O `var` é a forma antiga de declarar variáveis em JS. Ele tem **escopo de função** (não de bloco), o que pode causar comportamentos inesperados. Por isso, o uso de `var` é **desencorajado** no JavaScript moderno — prefira sempre `let` ou `const`.

---

## Boas Práticas de Declaração

Preferir `let` e `const` sobre `var` para melhorar o controle de escopo e prevenção de bugs.

> 💡 **Regra prática:** use sempre `const` por padrão. Só troque para `let` quando souber que o valor da variável vai mudar. Nunca use `var`.

---

## Conversão de Tipos

- `Number()` — Converte para número
- `String()` — Converte para texto
- `Boolean()` — Converte para verdadeiro/falso

> 💡 **Complemento:** Além da conversão explícita (acima), o JavaScript também faz **coerção implícita** — converte tipos automaticamente em certas operações. Exemplo:
> ```javascript
> "5" + 1   // "51"  → String concatenada
> "5" - 1   // 4     → Number convertido automaticamente
> Boolean(0) // false
> Boolean("") // false
> Boolean("texto") // true
> ```
> Isso pode causar bugs sutis, então é sempre melhor fazer conversões explícitas.
