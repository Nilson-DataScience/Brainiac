# Comandos Git

| Comando | Descrição |
|---|---|
| `git init` | Inicia o projeto |
| `git add` | Adiciona à área de preparação *(staging area)* |
| `git commit -m "mensagem"` | Para gravar as alterações |
| `git commit --amend -m "Nova Descrição"` | Altera a mensagem do último commit |
| `git status` | Mostra o estado atual do arquivo |
| `git clone` | Faz uma cópia do repositório remoto |
| `git log` | Verifica atualizações no código, mostrando um histórico de commits e seus IDs |
| `git push` | Para enviar as atualizações salvas para a nuvem |
| `git push --set-upstream origin "nome-branch"` | Define o branch remoto padrão para o push |
| `git pull` | Baixar mudanças do repositório remoto para o Git local |
| `git restore --source "id"` | Restaura os arquivos para o commit específico pelo seu ID |
| `git revert id` | Não apaga o commit — apenas cria um novo sem as alterações dos commits selecionados |
| `git reset --hard id-anterior-ao-deletado` | Apaga o commit e o remove do histórico |





> ⚠️ **Atenção com `git reset --hard`:** esse comando é **destrutivo** — apaga o commit do histórico permanentemente. Use com cuidado. Prefira `git revert` quando quiser desfazer sem perder o histórico.

## .gitignore

Arquivo usado para **não enviar** determinados arquivos/pastas para a nuvem.

**Exemplos de uso:**
```
nomeDaPasta/
nomeArquivo.extensão
*.extensão
```
