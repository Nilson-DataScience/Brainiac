# Branches na Prática

| Comando | Descrição |
|---|---|
| `git branch` | Lista todas as branches criadas no repositório |
| `git branch -m "NewName"` | Renomeia a branch |
| `git branch nome` | Cria uma branch que é uma ramificação da main |
| `git checkout nome` | Utilizado para acessar ou trocar de branch / main |
| `git branch -d nome` | Serve para deletar a branch relacionada, depois de ter juntado ela no main principal |
| `git branch -D nome` | Serve para apagar a branch mesmo sem ter juntado ela no main principal |
| `git merge nome-branch` | Utilizado para juntar os arquivos que estão separados por branch |
| `git log --oneline` | Consulta todos os logs de todas as branches |

| `git cherry-pick <id_commit>` | Consegue transferir o commit entre as ramificações |
| `git bisect bad` | Indica que o commit ainda está com bug |
| `git bisect good` | Indica que o commit está funcionando |
| `git bisect start` | Esse comando começa uma busca pelo bug |



> 💡 **Complemento — `git bisect`:** O bisect usa **busca binária** para encontrar qual commit introduziu um bug. O fluxo é: `git bisect start` → marcar um commit ruim com `git bisect bad` → marcar um commit bom com `git bisect good` → o Git vai dividindo o histórico até encontrar o commit problemático.
